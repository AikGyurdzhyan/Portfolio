# Проект: Подготовка ПМИ и руководства пользователя для сайта «Капсула»

## Цель проекта
Разработать программу и методику испытаний (ПМИ), а также руководство пользователя для сайта бронирования капсульных отелей «Капсула» версии 2.2, чтобы проверить исправленные ошибки и подтвердить соответствие продукта предъявленным требованиям.

## Выполненные задачи
- Составлен документ ПМИ, охватывающий все требования из плана тестирования:
  - Просмотр информации об отеле
  - Регистрация
  - Бронирование (авторизованным и неавторизованным пользователем)
  - Работа с личным кабинетом
  - Управление бронированиями
- Описаны предусловия, шаги проверки и ожидаемые результаты для каждого теста, включая ранее неисправленные ошибки.
- Уточнены сценарии регистрации:
  - Валидные данные
  - Незаполненные поля
  - Невалидные данные
  - Несовпадение паролей
  - Повторная регистрация
- Добавлены сценарии для проверки автозаполнения данных при бронировании авторизованным пользователем.
- Подготовлено руководство пользователя:
  - Пошаговые инструкции по просмотру информации об отелях, стоимости и отзывах
  - Регистрация и авторизация (вход, сброс пароля)
  - Редактирование персональных данных
  - Процедура бронирования без регистрации и с регистрацией
  - Управление бронированием (просмотр истории, отмена)
  - Описание типовых ошибок и способы их устранения

## Учёт бизнес-ограничений
- Доступ к сервису осуществляется через браузер Chrome версии 70+ на Windows или macOS.
- При бронировании все поля должны быть заполнены валидными данными; кнопка «Забронировать» неактивна при пустых или некорректных значениях.
- Личный кабинет автоматически создаётся для новых пользователей после первого бронирования.
- При ошибках регистрации и входа пользователю отображаются подсказки для исправления.

## Результат
- Подготовлена документация для проведения испытаний исправленной версии сайта «Капсула».
- Составлены подробные тестовые сценарии для проверки функционала.
- Создано руководство пользователя, обеспечивающее удобство клиентов и заказчика при проверке продукта.
- Документация готова к передаче заказчику для финальной приёмки продукта.

# Проект: Тестирование сайта «Капсула»

## Цель проекта
Провести функциональное тестирование веб-версии сервиса бронирования капсульных отелей «Капсула» (версия 2.2) и выявить дефекты, влияющие на работу ключевых пользовательских функций: регистрацию, личный кабинет, бронирование и управление бронированием.

## Выполненные задачи
- Разработан и дополнен план тестирования сайта, включая недостающие тест-кейсы.
- Протестированы основные функции:
  - Регистрация нового пользователя
  - Вход и работа в личном кабинете
  - Бронирование капсульного отеля авторизованным и неавторизованным пользователем
  - Управление бронированиями (просмотр истории и отмена)
- Задокументированы ожидаемые и фактические результаты по каждому тесту.
- Выявлены ошибки в функционале сайта:
  - Некликабельные символы отелей на карте, отсутствие всплывающих подсказок (**Capsule-1**);
  - Неинтерактивная карта, невозможность изменения масштаба (**Capsule-2**)
  - Отсутствие письма с подтверждением при регистрации (**Capsule-3**)
  - Не приходят письма с логином/паролем и подтверждением бронирования для новых пользователей (**Capsule-8**)
  - Невозможность редактирования email в личном кабинете (**Capsule-11**)
  - Невозможность отмены бронирования (**Capsule-12**)

## Учёт бизнес-ограничений
- Функции бронирования должны быть доступны как авторизованным, так и новым пользователям.
- Личный кабинет должен автоматически создаваться при первом бронировании неавторизованным пользователем.
- Все уведомления (регистрация, подтверждение бронирования, учётные данные) должны приходить на e-mail пользователя.
- Управление бронированиями (редактирование данных, отмена) должно быть доступно через личный кабинет.
